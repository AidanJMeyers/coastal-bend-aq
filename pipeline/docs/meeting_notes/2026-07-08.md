## 1. Decisions made

- **REDCap access should be API-only.** No more manual CSV exports.
  Reason: local file copies risk drift into OneDrive backups that
  could contain PII. Aidan: "if you're coding based on the API, then
  you automatically pull the updated data versus having to re-export
  every time." Dr. Warden: "we should exclusively use APIs."
- **Publish quarterly analytic files to the TAMU secure drive**
  (behind Texas's firewall) rather than distributing API tokens to
  students. Automate the quarterly refresh to run without manual
  intervention. Compare to the Isle of White Rush workflow.
- **API tokens only for people who already have REDCap access.**
  Undergrads + grad students without official delegation-log access
  use the published quarterly de-identified analytic files.
- **QC first, analysis second.** Only QC'd data should back any
  student project. Aidan is currently the only one doing QC and only
  on the initial form — needs to be delegated across the team.
- **Schedule a dedicated meeting for**:
  (a) QC procedures + delegation,
  (b) Quarterly analytic-file scope + format,
  (c) Target production of the first quarterly file at Month 6 of
      collection (~one month out).
  Meeting scheduled: **Tuesday next week (~2026-07-14), 2:00 PM Central / 3:00 PM Eastern.**
- **Wide vs long data format:** wide (with `month_1_`, `month_2_`,
  `month_3_` prefixes) — matches the Isle of White convention.
  REDCap defaults to long; conversion happens at analytic-file
  publication.
- **Withdraw participants 2 and 5** from BREATHE-CC cohort
  (never completed anything; flagged as forfeited).
- **Coastal Bend AQ pollutant data** will be pulled by BREATHE-CC
  students **via API** — same principle as REDCap; the fact that our
  Coastal Bend Neon schema is already API/SQL-accessible fits this
  workflow perfectly.
