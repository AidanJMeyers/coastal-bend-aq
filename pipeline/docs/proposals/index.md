## Why this section exists

- **External-collaborator friction.** When we outreach to Dr. Warden,
  Dr. Jin, or Jasmine's atmospheric-physics mentor, we need a single
  URL that says "here is the actual project we want to consult on" —
  meeting notes are too granular, pipeline updates are too process-y.
- **Grant reuse.** The scope docs can be lifted almost verbatim into
  grant applications (R25, climate-change seed grant resubmit, etc.)
  once they've been refined via the collaborator consult.
- **Team alignment.** Before spending Warden's + Jin's time, all three
  AQ leads should have signed off on the written scope.

## How to add a proposal

1. Create `proposals/<short_slug>.md` following the structure of
   [refinery_row_directional_health.md](./refinery_row_directional_health.md).
2. Add a row to the "Active proposals" table above.
3. Link the originating meeting notes.
4. Add a nav entry in `mkdocs.yml` under `Project proposals`.
5. Open a team review item on the [meeting dashboard](../meeting_notes/index.md).
